## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vite | 6.x | 构建工具 - 快速的开发服务器和打包工具 |
| React | 18.x | UI 框架 - 用于构建用户界面 |
| TypeScript | 5.x | 编程语言 - 提供类型安全 |
| TailwindCSS | 3.x | CSS 框架 - 快速编写样式 |

### 为什么选择这些技术?

**Vite vs Next.js**:
- Vite 启动速度更快(毫秒级)
- 配置更简单,学习曲线平缓
- 更适合初学者和小型项目
- 热更新(HMR)速度极快

**TailwindCSS**:
- 不需要写 CSS 文件
- 直接在 HTML 中使用类名
- 响应式设计简单
- 文件体积小

## 环境要求

在开始之前,请确保你的电脑上已安装:

- **Node.js** 18.0 或更高版本
  - 检查版本: `node --version`
  - 下载地址: https://nodejs.org/

- **npm** 或 **yarn** 包管理器
  - 检查版本: `npm --version`
  - npm 随 Node.js 一起安装

## 安装步骤

### 1. 进入前端目录

```bash
cd frontend
```

### 2. 安装依赖

```bash
npm install
```

这会安装所有必需的包,包括:
- `react` 和 `react-dom` - React 核心库
- `vite` - 构建工具
- `typescript` - TypeScript 编译器
- `tailwindcss` - CSS 框架
- `@vitejs/plugin-react` - Vite 的 React 插件

**安装时间**: 通常需要 10-30 秒(取决于网络速度)

## 启动项目

### 开发模式(推荐)

```bash
npm run dev
```

这会启动开发服务器,特点:
- ⚡ 热更新 - 修改代码后自动刷新
- 🔍 详细的错误提示
- 📊 实时的构建信息

**默认地址**: http://localhost:3002

如果端口被占用,Vite 会自动使用下一个可用端口(如 3001, 3002 等)

### 构建生产版本

```bash
npm run build
```

这会创建优化后的生产版本:
- 代码压缩
- Tree-shaking(删除未使用的代码)
- 输出到 `dist/` 文件夹

### 预览生产版本

```bash
npm run preview
```

在本地预览构建后的生产版本,确保一切正常。

## 常见问题

### Q1: 端口被占用怎么办?

**现象**:
```
Port 3000 is in use, trying another one...
Local: http://localhost:3001/
```

**解决方案**:
- Vite 会自动使用下一个可用端口
- 或者手动指定端口: `npm run dev -- --port 4000`

### Q2: 依赖安装失败

**现象**:
```
npm ERR! code ENOENT
```

**解决方案**:
1. 删除 `node_modules` 和 `package-lock.json`
   ```bash
   rm -rf node_modules package-lock.json
   ```
2. 重新安装
   ```bash
   npm install
   ```

### Q3: TypeScript 报错

**现象**:
```
Cannot find module '@/types/task'
```

**解决方案**:
- 检查 `tsconfig.json` 中的路径配置
- 确保 `vite.config.ts` 中配置了 alias
- 重启开发服务器

### Q4: 页面是空白的

**原因**:
1. 检查浏览器控制台是否有错误
2. 确保所有文件都已保存
3. 尝试清除浏览器缓存(Ctrl + Shift + R)

### Q5: 修改代码后页面没有更新

**解决方案**:
1. 检查文件是否保存
2. 重启开发服务器(Ctrl + C 然后重新运行 `npm run dev`)
3. 清除浏览器缓存
